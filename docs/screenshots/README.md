# Screenshots

This folder holds screenshots referenced by the main [README](../../README.md).

## Suggested shots to capture

Drop PNGs here using these filenames so the README references line up:

| Filename | What it should show |
|---|---|
| `hero.png` | The whole DVSA page with the floating pill + gear visible bottom-right. Best taken on a logged-in state with a real test date showing. |
| `settings-panel.png` | The full settings panel, scroll to fit, or split into two: `settings-panel-top.png` (Health + Date window + monitoring preview) and `settings-panel-bottom.png` (Auto-book + Instructor + Advanced). |
| `status-pill.png` | Close-up of the floating pill while scanning, ideally with a countdown visible. |
| `monitoring-preview.png` | The "What you're monitoring" card with a real breakdown bar visible. |
| `history-panel.png` | The history modal with several findings and the KPI tile grid populated. |
| `alert-fired.png` | The red banner overlay when a match fires, with the Show me button visible. |

## Tips

- **Crop tight**, only include relevant UI.
- **Hide personal data**, blur or remove your licence number, booking ref, real name on the page, and full test centre if you want extra privacy.
- **Use a clean profile**, fewer browser extensions visible = cleaner shot.
- **Light mode**, the script is styled for light backgrounds; dark-mode screenshots will look out of place against the DVSA white-on-blue theme.

Once you've added the PNGs, update the `<!-- TODO: -->` markers in the README with proper `![alt](docs/screenshots/filename.png)` references.
